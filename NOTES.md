# Editorial Notes

These notes keep the source material and maintenance rules for this repository. The repository itself is English-only.

## Purpose

This repository manifests public evidence of Openapi SpA's certifications, recognitions, memberships, registrations, patent/trademark information, and certified partner ecosystem.

## Canonical Public Sources

- Openapi public website: https://openapi.com
- Certifications, special recognitions, memberships, registrations, and patents: https://openapi.com/certifications
- Certified partner directory: https://openapi.com/search-partner
- Public system status page: https://openapi-com.statuspage.io
- Privacy domain: https://privacy.openapi.com
- Legal documents domain: https://legal.openapi.com

## Content Kept

The README keeps information that is relevant to the repository purpose:

- Certifications and authorizations published by Openapi.
- Special recognitions and legality rating.
- Memberships, including the Linux Foundation Silver Member note.
- Public registrations.
- SmartFire patent and trademark note.
- Certified partner directory summary.
- Company identity information from the public footer.

## Content Removed From The Raw Notes

The original notes contained a copied website footer. The following material was intentionally not carried into the README because it is navigation, UI, commerce, or operational footer content rather than certification or partnership evidence:

- Header and footer navigation links.
- Language selector markup.
- Social media links.
- Payment method badges.
- Cookie preference link.
- Repeated image, picture, and CSS markup.
- Generic product, pricing, sitemap, and contact navigation.
- Raw HTML copied from the public site.

## Normalization Rules

- Use "Openapi" for the company brand as it appears on the public website.
- Use "Openapi SpA" when referring to the legal entity.
- Keep certification names in their official form, including standard numbers and publication years.
- Keep certificate, licence, registration, and authorization identifiers when publicly available.
- Avoid marketing language unless it is needed to explain the public status.
- Do not claim that a certification is active beyond what can be verified from public sources.
- When counts change, update both the README and this notes file after checking the public source.

## Maintenance Checklist

Before changing the public README:

1. Check https://openapi.com/certifications.
2. Check https://openapi.com/search-partner if partner counts, levels, or languages are mentioned.
3. Keep only public and sourceable information.
4. Keep the writing in English.
5. Remove copied HTML and duplicated footer material from notes.
